PhishGuard

"PhishGuard: Phishing Simulation and Awareness Platform"

Problem Statement

PhishGuard is a platform that simulates real-world phishing attacks to assess user vulnerability and improve cybersecurity awareness through interactive training and analytics.

Literature Review / Market Research

  • Phishing Simulation Exercise in a Large Hospital: A Case Study (2022)
  • Anti-Phishing Training Does Not Work: A Large-Scale Empirical Study (2010, updated 2024 research)
  • Evaluating Organizational Phishing Awareness Training (2023)
  • The Effects of Group Discussion and Role-Playing Training on Self-Efficacy and Vigilance (2024)
  • Phishing Detection Using Machine Learning Based URL Analysis (2021)

Research Gap / Innovation

Lack of Realism: Existing training is theoretical and fails to simulate real-world attacks.
High Costs: Enterprise solutions are expensive for educational institutions and SMBs.
Technical Barriers: Open-source tools require advanced expertise.
No Immediate Feedback: Lacks real-time, contextual guidance.

System Methodology

Model / Architecture

Algorithm Overview

  • User Registration: Users register and are assigned roles.
  • Campaign Creation: Admin creates phishing campaigns using templates.
  • Email Simulation: System generates and sends phishing emails.
  • Behavior Tracking: Tracks opens, clicks, and submissions.
  • Risk Analysis: Calculates user risk scores based on patterns.
  • Instant Training: Provides immediate feedback.
  • Reporting: Generates analytics and insights.

System Workflow

  • Frontend: Interactive dashboard with WebSockets.
  • Backend: Flask/Django APIs with Celery + Redis.
  • Database: MySQL for storage, Redis for caching.
  • Infrastructure: Dockerized with Kubernetes scaling.
  • Security: TLS, JWT, CSRF, and SQLi protection.
